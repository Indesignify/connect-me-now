User case 1. Регистрация пользователя.
Пользователь открывает сайт и видит главное окно с тремя кнопками: "Авторизация", "Регистрация", "Где я?"
Внизу страницы находится кнопка, открывающая информацию о проекте.
Пользователь нажимает кнопку "Регистрация", открывается окно с формой для заполнения. Поля формы: "Имя", "Фамилия", "E-mail", "Username", "Password", "Confirm password". Также внизу формы находятся кнопки, позволяющие зарегистрироваться с помощью личных данных из других социальных сетей (ВКонтакте, Facebook, GitHub).
После заполнения основной формы регистрации на указанный адрес электронной почты приходит письмо с ссылкой, позволяющей завершить регистрацию. В окне приложения всплывает сообщение о том, что необходимо пройти по ссылке в письме, которое отправлено на указанный электронный адрес.
Пользователь переходит по ссылке в письме и открывается главное окно приложения.

User case 2. Авторизация и поиск друзей.
Пользователь нажимает кнопку "Авторизация", расположенную на стартовой странице приложения. Появляется окно с формой "логин-пароль". Внизу формы расположены кнопки, дающие возможность авторизоваться через учётные записи популярных социальных сетей (ВКонтакте, Facebook, GitHub), если регистрация происходила при их помощи. 
При многократном неверном вводе логина и/или пароля выводится автоматическая проверка-антибот (капча).
Если пользователь корректно ввёл данные, то открывается основная страница приложения. При первой авторизации предлагается найти друзей, зарегистрированных в данном приложении, при помощи E-mail или социальной сети, указанной при регистрации. Имеется возможность просмотреть профили друзей, а также добавить их во френдлист.

User case 3. Использование приложения без регистрации.
Пользователь нажимает кнопку "Где я?", расположенную на стартовой странице приложения. Открывается интегрированная в приложение карта (Google). Пользователю через всплывающее уведомление даётся возможность разрешить доступ к его местоположению. Далее, в случае согласия пользователя появляется метка на карте. В противном случае карта открывается в случайном месте и появляется уведомление о том, что нет возможности установить местоположения пользователя без разрешения.

User case 4. Использование пользователем полоски меню.
Пользователь открывает приложение, авторизуется. Открывается карта, над ней расположено полоска меню с кнопками. Кнопки меню: "Друзья", "Добавить друга", "Настройки", "Мой профиль", "Выход".
При нажатии на кнопку "Друзья" в левой части экрана появляется столбик со списком друзей. Друзья могут иметь 3 статуса: "Онлайн", "Не беспокоить", "Оффлайн". Если пользователь включил статус "Не беспокоить", то ему невозможно отправить запрос на встречу. Если друг из списка имеет статус "Онлайн", то возможно соединение пользователей (User case 6).
При нажатии на кнопку "Добавить друга" всплывает окно с четырьмя вкладками: "E-mail", "ВКонтакте", "Facebook", "GitHub". При нажатии на любую из этих вкладок появляется поле для ввода информации, необходимой для поиска друга. Поиск осуществляется по нажатию кнопки "Найти". При нахождении зарегистрированных пользователей, удовлетворяющих критериям поиска, их список выводится в этом же окне. У пользователя есть возможность просмотреть профили найденных людей, а также отправить запрос на добавление в друзья.  
При нажатии на кнопку меню "Настройки" открывается окно с настройками: есть возможность изменить свой статус ("Онлайн"/"Не беспокоить"), а также включить/выключить звуковые оповещения о желании друга подключиться. Кроме того, пользователь может скрыть свои личные данные (имя и фамилию).
При нажатии на кнопку меню "Мой профиль" открывается окно с профилем пользователя. На странице отображены аватар, никнейм, имя, фамилия и почта/ссылка на социальную сеть (в зависимости от способа регистрации пользователя). У пользователя есть возможность изменить любые из указанных выше данных.
При нажатии на кнопку "Выход" всплывает окно с требованием подтверждения выхода из аккаунта. Если пользователь соглашается, то происходит выход из учётной записи и открывается главная страница приложения.

User case 5. Просмотр профилей других пользователей.
Пользователь открывает приложение, авторизуется. Открывается окно с картой, сверху расположена полоска меню. У пользователя (A) есть два способа открыть профиль другого пользователя (B): при помощи поиска (User case 4) или открыть профиль друга, выбранного из списка друзей. В профиле пользователя B пользователь A может увидеть следующую информацию: аватар, никнейм, имя и фамилия (если пользователь B в настройках разрешил отображать имя и фамилию). Если пользователь B не находится в друзьях у пользователя A, то пользователь A может отправить запрос на добавление в друзья пользователю B при помощи кнопки "Добавить в друзья". Если пользователь B находится в друзьях у пользователя A, то пользователь A может удалить пользователя B из своего списка друзей при помощи кнопки "Удалить из друзей" (у пользователя B также удалится из списка друзей пользователь A).

User case 6. Использование приложения зарегистрированным пользователем: сценарий встречи с другом.
Пользователь открывает приложение, авторизуется. Открывается окно с картой, сверху находится полоска меню. При нажатии на кнопку "Друзья" в левой части экрана появляется столбик со списком друзей. Пользователь (A) выбирает онлайн друга (B) из списка и отправляет ему запрос на встречу. У пользователя B появляется оповещение (со звуком) о желании пользователя A соединиться. Если пользователь B отказывается от встречи, то пользователь A получает уведомление об этом на экране приложения. Если пользователь B принимает приглашение, то у обоих пользователей на карте появляются метки, соответствующие местоположению обоих пользователей, а также прокладывается оптимальный маршрут между ними. При нажатии кнопки "обновить карту" происходит обновление метки пользователя-друга в соответствии с его новым местоположением. Также возможна перестройка оптимального маршрута, если алгоритм найдёт более короткий путь. Когда два пользователя встречаются (либо отменяют встречу), любой из пользователей может нажать на кнопку "disconnect", после чего соединение между двумя пользователями будет прекращено.
